隐式类型转换

隐式类型转换

  • 使用===和!==而不是==和!=(避免隐式类型转换带来的问题)
    反模式示例:
      let string = '';
  let string2 = false;
  string == string2 && doSomething();
  string != string2 && doSomething2();
    最佳实践:
      let string = '';
  let string2 = false;
  string === string2 && doSomething();
  string !== string2 && doSomething2();
    或者使用lodash库的isEqual方法作比较
      import isEqual from 'lodash/isEqual';
  let string = '';
  let string2 = false;
  isEqual(string,string2) && doSomething();
  !isEqual(string,string2) && doSomething2();
    eslint规则: 
      "eqeqeq":"warn"
  • 如果需要做比较需要强制转换成布尔类型，转换遵循以下规则
    1. 对象=>true
    2. undefined => false
    3. null => false
    4. boolean => boolean
    5. number => 0或NaN => false 其它 => true
    6. string => string.trim() === '' => false 其它 => true
  • 抽象相等比较x == y ecmascript规范7.2.14
    1. 如果x的数据类型与y的数据类型一样,返回结果为:x === y;
    2. 如果x的值为null,y的值为undefined,返回结果为:true;
    3. 如果x的值为undefined,y的值为null,返回结果为:true;
    4. 如果x的数据类型为Number,y的数据类型为String,返回结果为:x == Number(y);(文档上是x == !Number(y),这儿应该是文档有误)
    5. 如果x的数据类型为String,y的数据类型为Number,返回结果为:Number(x) == y;(文档上是!Number(x) == y,这儿应该是文档有误)
    6. 如果x的数据类型为Boolean,返回结果为:Number(x) == y;(同上文档上取反)
    7. 如果y的数据类型为Boolean,返回结果为:x == Number(y);(同上文档上取反)
    8. 如果x的数据类型为String、Number,或者Symbol,y的数据类型为Object,返回结果为:x == ToPrimitive(y);
    9. 如果x的数据类型为Object,y的数据类型为String、Number,或者Symbol,返回结果为:ToPrimitive(x) == y;
    10. 其它情况返回:false;
  • 强转为数字类型Number(x) ecmascript规范7.1.3
    1. x的数据类型为Undefined,返回结果为:NaN;
    2. x的数据类型为Null,返回结果为:0;
    3. x的数据类型为Boolean,如果x的值等于true返回结果为:1、如果x的值等于false返回结果为:0;
