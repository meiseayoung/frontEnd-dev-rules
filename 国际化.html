国际化

国际化

  • 方案一(集权管理单文件)
    在src目录下新建i18n文件夹,然后新建index.js
    代码示例: 

      export default {
    ch: {//中文
      MESSAGE: '消息'
    },
    en: {//英语
      MESSAGE: 'Message'
    },
    ge: {//德语
      MESSAGE: 'Nachrichten'
    },
    ...
  }

    优点:

    • 一次引入所有国际化内容
    • 动态切换语言方便

    缺点:

    • 国际化内容太多时单个文件太大
    • 不利于更改国际化内容(文件太大查找引索不方便)
    • 不利于按需载入

  • 方案二(集权管理多文件) 在src目录下新建i18n文件夹,然后针对每个语言新建一份国际化文件
    代码示例:

    1. ch.i18n.js 
      export default {
 MESSAGE: '消息'
}
    2. en.i18n.js 
      export default {
 MESSAGE: 'Message'
}
    3. ge.i18n.js 
      export default {
 MESSAGE: 'Nachrichten'
}

    优点:

    • 方便按需载入
    • 更改国际化内容方便

    缺点:

    • 动态切换语言不太方便
    • 组件使用国际化内容时需要先查找是否已经已经在国际化文件中定义

  • 方案三(分布式管理)
    每个组件各自负责自己需要的国际化内容
    代码示例:
    Icon.i18n.js 

      export default {
    ch: {//中文
      MESSAGE: '消息',
      ICON: '图标'
    },
    en: {//英语
      MESSAGE: 'Message',
      ICON: 'Icon'
    },
    ge: {//德语
      MESSAGE: 'Nachrichten',
      ICON: 'symbole'
    },
    ...
  }

    Button.i18n.js 

      export default {
    ch: {//中文
      MESSAGE: '消息',
      BUTTON: '按钮'
    },
    en: {//英语
      MESSAGE: 'Message',
      BUTTON: 'Button'
    },
    ge: {//德语
      MESSAGE: 'Nachrichten',
      BUTTON: 'knopf'
    },
    ...
  }

    优点:

    • 方便按需载入
    • 更改国际化内容方便
    • 动态切换语言方便

    缺点:

    • 不同组件使用到的国际化内容相同时重复定义了

  • 方案四(分布式配合集中式管理)
    实现与方案三实现一致，不过在此基础上还需要合并所有组件下的国际化文件,相同国际化内容分配同一个key
    合并后的文件 

      export default {
    ch: {//中文
      'rb93q3otg3g': '消息',
      'qilqin642d': '按钮',
      '6j7mep13oo8': '图标'
    },
    en: {//英语
      'rb93q3otg3g': 'Message',
      'qilqin642d': 'Button',
      '6j7mep13oo8': 'Icon'
    },
    ge: {//德语
      'rb93q3otg3g': 'Nachrichten',
      'qilqin642d': 'knopf',
      '6j7mep13oo8': 'symbole'
    }
  }

    优点:

    • 方便按需载入
    • 更改国际化内容方便
    • 动态切换语言方便

    缺点:

    • 合并文件实现技术相对其它方案来说复杂
